# 农场游戏全系统数值梳理与审查报告

**审查日期**: 2025-06-27  
**审查标准**: 专业数值策划审计框架（交叉验证、边界检查、一致性审查、对标分析）  
**核心设定**: 996打工人回到乡下种田过活（个体小农户，不雇佣工人，不机器自动化）  

---

## 一、审查方法论

作为资深数值策划，本次审查遵循以下原则：
1. **数据一致性**：所有数据对象之间的交叉引用是否一致
2. **边界值检查**：极限条件下的数值是否合理（负数、溢出、除零）
3. **经济平衡**：收入/支出/成长曲线是否可持续
4. **体验节奏**：操作频率、等待时间、反馈周期是否符合"放置+治愈"定位
5. **文案统一**：术语一致性（元/金币、斤/kg等）

**数据完整性**：✅ 4种作物，字段完整

| 作物 | 种子价 | 产量 | 单价 | 生长天数 | 净利润 | 日收益 | 季节 |

## 四、跨系统交叉验证

### 4.1 经济循环验证

```
收入端：
  - 作物售卖：早稻600元/周期（28分钟1x）
  - 建筑收入：鸡舍15元/天（33天回本）
  - 采集收入：平均15-22元/次
  - 加工收入：碾米ROI 17%

支出端：
  - 种子：60-90元/次
  - 租金：100-400元/季（60天）
  - 工具：50-100元/件
  - 建筑：500-5000元/座
  - 宠物食物：8-10元/天

结论：
  - 早期（1-7天）：日收入约50-100元，勉强覆盖租金+种子
  - 中期（7-30天）：日收入约200-500元，开始积累
  - 后期（30天+）：日收入约1000+元，财务自由
  - 整体经济循环：可持续，但前期较紧
```

### 4.2 时间系统验证

```
1倍速：2.5秒真实时间 = 1小时游戏时间
1游戏天 = 60秒真实时间 = 1分钟
1游戏季（60天）= 60分钟真实时间 = 1小时
1游戏年（240天）= 4小时真实时间

问题：
  - 1分钟就过一天，时间流速过快
  - 1年就4小时，缺乏长期经营感
  - 建议：1倍速改为5秒/小时（1天=2分钟）
```

### 4.3 离线收益验证

```
当前：离线72游戏小时 = 180秒真实时间 = 3分钟
标准：放置游戏离线12-24小时
差距：3分钟 vs 12小时 = 240倍

问题：
  - 玩家去倒杯水回来就满收益
  - 完全没有"挂机"体验
  - 建议：离线上限改为12小时真实时间
```

### 4.4 体力系统验证

```
新手房：每小时恢复5点（20/4），每天120点
体力上限：120点
结果：每天正好恢复满 → 无限操作

问题：
  - 放置游戏核心体验"等待恢复"缺失
  - 建议：上限200，日恢复80（或上限120，日恢复60）
```

### 核心问题一句话总结

> **当前游戏是"需要持续操作的模拟经营"，不是"放置挂机游戏"。**
>
> 三个致命问题：
> 1. **离线3分钟就满收益**（标准应为12小时）
> 2. **早稻28分钟一个周期**（标准应为5分钟）
> 3. **体力永远不缺**（新手房每天正好回满）
>
> 修复后，游戏将成为"上班前种一波，午休收一波，下班再种"的真正放置体验。
