## 一、本轮发现并修复的问题（共 8 个）

### 🔴 致命 Bug 1: `constants.js` 重复定义覆盖
**文件**: `js/constants.js`  
**问题**: ECONOMY/REPUTATION/PRESSURE/INVENTORY 被定义两次，旧版数值覆盖新版。  
**影响**: 初始金币3250（应为1500）、声望1.0（应为0.5）、压力10（应为20），数值平衡完全失效。  
**修复**: 删除旧版重复定义（第74-135行、368-375行）。

### 🔴 致命 Bug 2: `patients.js` `_generateOwner` 重复定义
**文件**: `js/patients.js`  
**问题**: 旧版随机名字池函数覆盖了新版NPC系统函数。  
**影响**: ownerType 不是有效NPC ID，所有NPC头像、羁绊、对话系统失效。  
**修复**: 删除旧版 `_generateOwner`（第130-144行）。

### 🔴 致命 Bug 3: `main.js` 缺失三个治疗函数
**文件**: `js/main.js`  
**问题**: `doMedicineTreatment()`、`doHerbTreatment()`、`doBuyTreatment()` 被引用但完全未定义。  
**影响**: 点击任何治疗选项按钮无任何反应，治疗流程中断。  
**修复**: 补充三个函数（第502-575行），含库存检查、资源扣除、成功率计算、finishTreatment调用。

### 🔴 致命 Bug 4: `core.js` `startNewDay` 重复代码
**文件**: `js/core.js`  
**问题**: 破产检测代码重复（第415-418行），`_bankruptDays` 被加两次。  
**影响**: 连续2天金币<0就触发破产（应为3天）。  
**修复**: 删除重复代码块。

### 🟡 视觉问题: 角色/道具/迷你游戏素材白底
**文件**: 50+ 个 PNG 素材  
**问题**: 所有素材扩展名为 `.png`，但实际格式为 JPEG（无透明通道）。  
**影响**: 角色立绘、道具图标、迷你游戏元素在深色背景上显示白色矩形背景。  
**修复**: Python Pillow 批量处理 50 个素材文件，将白色背景设为透明，保存为真正 PNG。

### 🟡 视觉问题: UI 容器白色背景
**文件**: `pet_doctor_v0.3.html`  
**问题**: 对话气泡、道具栏、等待队列、结算卡片使用白色半透明背景。  
**影响**: 在深色游戏主题中显得突兀。  
**修复**: 改为深色半透明背景 + 浅色文字。

### 🟡 代码问题: `ui.js` `getNPCAvatar` 重复定义
**文件**: `js/ui.js`  
**问题**: 第二个重复定义逻辑错误（`!npc` 时 `npc` 为 falsy，导致 `npc.avatar` 永远不会执行）。  
**影响**: NPC 心情头像无法正确映射。  
**修复**: 删除第二个重复定义。

### 🟡 健壮性: 图片缺失导致 broken image
**文件**: `pet_doctor_v0.3.html`  
**问题**: 9 个 NPC 头像素材缺失，加载时显示 broken image 图标。  
**修复**: 添加全局 `img` 错误处理，缺失时显示 emoji 占位并隐藏错误图标。
