## 一、本次测试中发现并修复的问题

### 🔴 致命 Bug 1: `constants.js` 重复定义覆盖
**问题**: ECONOMY/REPUTATION/PRESSURE/INVENTORY 被定义两次，旧版数值覆盖新版。  
**影响**: 初始金币3250（应为1500）、声望1.0（应为0.5）、压力10（应为20）。数值平衡完全失效。  
**修复**: 删除旧版重复定义（第74-135行、368-375行）。

### 🔴 致命 Bug 2: `patients.js` `_generateOwner` 重复定义
**问题**: 旧版随机名字池函数覆盖了新版NPC系统函数。  
**影响**: ownerType 不是有效NPC ID，所有NPC头像、羁绊、对话系统失效。  
**修复**: 删除旧版 `_generateOwner`（第130-144行）。

### 🔴 致命 Bug 3: `main.js` 缺失三个治疗函数
**问题**: `doMedicineTreatment()`、`doHerbTreatment()`、`doBuyTreatment()` 被引用但**完全未定义**。  
**影响**: 点击任何治疗选项按钮无任何反应，治疗流程中断。  
**修复**: 在 `main.js` 第502-575行补充三个函数，含库存检查、资源扣除、成功率计算、finishTreatment调用。

### 🟡 视觉问题: 角色立绘/道具白底
**问题**: 所有素材文件扩展名为 `.png`，但实际格式为 JPEG（无透明通道）。  
**影响**: 角色立绘、道具图标在深色背景上显示白色矩形背景。  
**修复**: 用 Python Pillow 批量处理 44 个素材文件，将角落背景色设为透明，保存为真正 PNG。  
**处理文件**: 医生立绘×5、宠物立绘×16、NPC头像×11、道具×13。

### 🟡 视觉问题: 对话气泡/道具栏/等待队列白色背景
**问题**: CSS 中使用 `rgba(255,255,255,0.96)` 等白色半透明背景。  
**影响**: 在深色游戏主题中显得突兀。  
**修复**: 将 `.dialogue-inner`、`.item-slot`、`.waiting-pet`、`.summary-card` 改为深色半透明背景，文字颜色改为浅色。
