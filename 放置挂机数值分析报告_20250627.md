# 农场游戏放置挂机数值体系分析报告

**分析日期**: 2025-06-27  
**分析标准**: 放置游戏(Idle Game)设计方法论 - 参照《Adventure Capitalist》《Melvor Idle》《Cookie Clicker》等标杆产品

---

## 一、放置游戏核心设计原则（行业标准）

根据GDC演讲《Idle Games: Mechanics and Monetization》及行业实践，放置游戏必须满足以下核心机制：

### 1.1 离线收益系统（Offline Progression）
| 参数 | 行业标准 | 说明 |
|------|---------|------|
| 离线上限 | 12-24小时真实时间 | 让玩家可以安心睡觉/上班 |
| 收益计算 | 基于离线时长×当前产出率 | 上线即有明显收获感 |
| 上限机制 | 软上限（收益递减）或硬上限 | 防止无限积累 |
| 召回效果 | 离线越久，回归动力越强 | 73%日活用户每天回归2次以上 |

### 1.2 成长曲线设计（Growth Curve）
| 类型 | 公式 | 用途 |
|------|------|------|
| 收入 | 线性/多项式增长 | income_total = base × owned |
| 成本 | 指数增长 | cost_next = base × rate^owned |
| 结果 | 成本增长 > 收入增长 | 持续产生挑战感 |

### 1.3 时间管理（Time Management）
| 玩家类型 | 在线时长 | 获得收益比例 | 体验设计 |
|---------|---------|------------|---------|
| 轻度 | <1小时 | 30% | 登录领奖即可 |
| 中度 | 1-2小时 | 80% | 需要一定操作 |
| 重度 | 3-4小时 | 100% | 完整体验 |
| 核心 | >5小时 | 100%+周限奖励 | 需要挂机/策略 |

### 1.4 循环周期设计（Loop Design）
- **早期**：5-15分钟一个完整循环（种植→收获→售卖）
- **中期**：30分钟-2小时一个循环
- **后期**：数小时一个循环，配合自动化减少操作

## 二、当前游戏数值分析

### 2.1 时间系统

| 参数 | 数值 | 换算 |
|------|------|------|
| 1倍速tick间隔 | 2.5秒真实时间 | — |
| 每个tick | 1小时游戏时间 | — |
| 1游戏天 | 60秒真实时间 | =1分钟 |
| 1游戏季(60天) | 60分钟真实时间 | =1小时 |
| 1游戏年(240天) | 4小时真实时间 | — |

**分析**：时间流速过快，1分钟就过一天，缺乏"放置"感。

### 2.2 作物生长周期

| 作物 | 游戏天数 | 1倍速真实时间 | 5倍速真实时间 |
|------|---------|--------------|--------------|
| 早稻 | 28天 | 28分钟 | 5.6分钟 |
| 晚稻 | 25天 | 25分钟 | 5.0分钟 |
| 春薯 | 65天 | 65分钟 | 13.0分钟 |
| 秋薯 | 55天 | 55分钟 | 11.0分钟 |

**分析**：
- 早稻28分钟一个循环，**远超**放置游戏早期5-15分钟标准
- 即使5倍速也需要5.6分钟，玩家需要一直盯着屏幕
- 春薯13分钟，不符合"上班摸鱼"的碎片化体验

### 2.3 离线收益系统

| 参数 | 数值 | 问题 |
|------|------|------|
| 离线→游戏时间换算 | 1秒真实 = 0.4游戏小时 | 转换率低 |
| 离线上限 | 72游戏小时 | 相当于3分钟真实时间 |
| 实际效果 | 离线3分钟就满 | **严重不足** |

**分析**：玩家去倒杯水回来就满收益了，完全没有"挂机"体验。

### 2.4 体力系统

| 房屋 | 每小时恢复 | 每天恢复 | 体力上限 | 问题 |
|------|----------|---------|---------|------|
| 新手房 | 5点 | 120点 | 120 | **正好回满** = 无限操作 |
| 进阶房 | 6.25点 | 150点 | 120 | 溢出浪费 |
| 精英房 | 8.75点 | 210点 | 120 | 大量溢出 |
| 终极房 | 12.5点 | 300点 | 120 | 严重溢出 |

**分析**：
- 新手房每天正好恢复120点，等于体力上限
- 这意味着玩家**永远不会缺体力**，失去了"等待恢复"的放置体验
- 进阶房以上恢复大量溢出，设计完全失效

### 2.5 经济系统

| 作物 | 种子成本 | 产量 | 单价 | 总收入 | 净利润 | 日收益 |
|------|---------|------|------|--------|--------|--------|
| 早稻 | 60 | 200kg | 3.3 | 660 | 600 | 21.4 |
| 晚稻 | 70 | 180kg | 3.3 | 594 | 524 | 21.0 |
| 春薯 | 80 | 580kg | 1.5 | 870 | 790 | 12.2 |
| 秋薯 | 90 | 420kg | 1.5 | 630 | 540 | 9.8 |

**分析**：
- 收入是**固定线性**，没有指数增长
- 没有"升级产出"的复利效应
- 没有自动化升级（如自动浇水、自动收割）
- 每次循环收益相同，缺乏成长感

## 四、识别的数值漏洞（Critical Issues）

### 🔴 漏洞1：离线收益上限过低（最严重）
```
现状：离线3分钟 = 72游戏小时 = 上限
标准：离线12-24小时 = 上限

影响：玩家无法"上班摸鱼"，必须持续在线操作
修复建议：将离线上限改为12小时真实时间（= 720游戏天）
```

### 🔴 漏洞2：作物周期过长
```
现状：早稻28分钟/周期
标准：放置游戏初期5-15分钟/周期

影响：玩家需要长时间盯着屏幕，无法碎片化体验
修复建议：缩短为3-5分钟/周期（1倍速），或增加"加速生长"机制
```

### 🔴 漏洞3：体力系统失效
```
现状：新手房每天正好恢复120点 = 体力上限
标准：每天恢复量应 < 上限，迫使玩家等待或做决策

影响：玩家无限体力，失去了"等待恢复"的放置核心体验
修复建议：
- 方案A：体力上限提升，恢复速度降低（如上限200，每天恢复120）
- 方案B：体力恢复只在睡觉时进行，白天操作消耗体力
```

### 🔴 漏洞4：无指数增长曲线
```
现状：每次循环收益固定（600元/早稻）
标准：每次循环收益应随进度提升（如科技、升级、自动化）

影响：玩家长期没有成长感，30分钟后就觉得无聊
修复建议：引入升级系统：
- 土地升级：每次升级增加10%产量，成本指数增长
- 工具升级：解锁自动浇水/自动收割
- 技能加成：随时间提升收益倍率
```

### 🟡 漏洞5：时间流速过快
```
现状：1分钟真实时间 = 1天游戏时间
标准：放置游戏通常让时间有"重量感"

影响：1年就4小时，玩家没有长期经营感
修复建议：降低时间流速，或增加"时间冻结"选项
```

### 🟡 漏洞6：缺乏自动化系统
```
现状：所有操作必须手动（整地、播种、浇水、收割、售卖）
标准：放置游戏核心是"解锁自动化，减少操作"

影响：玩家需要持续点击，违背放置游戏初衷
修复建议：
- 解锁雇佣工人自动浇水
- 解锁自动收割（成熟后自动收入仓库）
- 解锁自动售卖（仓库满后自动卖出）
```

## 五、重新规划数值体系建议

### 5.1 目标定位重塑："办公摸鱼放置农场"

核心体验公式：
```
上班前：种下作物
工作中：偶尔点开看看进度
午休时：收割一波，升级一下
下班时：查看一天收益，再种一波
```

### 5.2 离线收益重构（最关键）

```javascript
// 当前代码（问题：3分钟上限）
const maxGameHours = 72; // 72游戏小时 = 3分钟真实时间

// 建议代码（12小时真实时间上限）
const maxRealHours = 12; // 12小时真实时间
const maxGameHours = maxRealHours * 3600 * 0.4; // 17280游戏小时 = 720游戏天
// 或更直接：按离线秒数计算上限
const maxOfflineSeconds = 12 * 3600; // 43200秒 = 12小时
```

离线收益内容扩展：
- 作物自动生长（离线期间作物正常生长）
- 自动体力恢复（离线期间体力恢复）
- 随机事件触发（离线期间有概率触发事件）
- 工人自动工作（雇佣工人离线期间继续产出）

### 5.3 作物周期重构

| 阶段 | 作物 | 周期 | 离线生长 | 设计意图 |
|------|------|------|---------|---------|
| 初期 | 小白菜 | 2分钟 | 支持 | 快速反馈，入门体验 |
| 初期 | 早稻 | 5分钟 | 支持 | 主要作物，第一次完整体验 |
| 中期 | 晚稻 | 10分钟 | 支持 | 升级后的选择 |
| 中期 | 春薯 | 15分钟 | 支持 | 高收益但慢 |
| 后期 | 秋薯 | 30分钟 | 支持 | 后期深度 |

### 5.4 体力系统重构

```javascript
// 建议：体力上限 > 日恢复，形成"等待"体验
const STAMINA_CONFIG = {
    maxStamina: 200,        // 上限提升

    autoWater: { unlockDay: 5, cost: 500, effect: '每天自动浇水一次' },
    autoHarvest: { unlockDay: 15, cost: 2000, effect: '成熟后自动收割' },
    autoSell: { unlockDay: 30, cost: 5000, effect: '仓库满后自动售卖' },

```

### 5.6 短期与长期目标设计

**短期目标（每次登录）**：
- 5分钟内：完成一次种植→收割→售卖循环
- 获得即时金币反馈
- 检查随机事件

**中期目标（每天）**：
- 升级一块土地/购买一个工具
- 解锁一个新功能（副业/自动化）
- 完成当日任务

**长期目标（每周）**：
- 升级房屋
- 解锁全部自动化
- 积累第一桶金（10000元）

## 七、结论

### 当前游戏本质
**这是一个"需要持续操作的模拟经营游戏"，而非"放置挂机游戏"。**

### 核心差距
| 维度 | 放置游戏标准 | 当前游戏 | 差距 |
|------|------------|---------|------|
| 离线体验 | 12小时离线收益 | 3分钟离线收益 | **240倍** |
| 循环周期 | 5-15分钟 | 28分钟 | **2-5倍** |
| 操作频率 | 5-10分钟一次 | 持续操作 | 不符合 |
| 成长曲线 | 指数增长 | 固定线性 | 无增长感 |
| 自动化 | 核心机制 | 完全缺失 | 完全缺失 |

### 建议方向
**方案A：轻量修改（快速上线）**
- 将离线上限改为12小时
- 缩短作物周期至5分钟
- 调整体力上限为200，日恢复100
- 增加自动浇水/收割解锁

**方案B：深度重构（长期体验）**
- 完全重做离线收益系统
- 引入土地升级+指数成本
- 添加自动化解锁树
- 降低时间流速，增加沉浸感

**推荐**：先实施方案A（约3小时），验证后再做方案B。
