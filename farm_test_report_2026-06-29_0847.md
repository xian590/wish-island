## 四、Bug 发现与修复

### Bug 1: 任务追踪器"待完成"数量统计错误

**严重程度**: 🟡 中  
**状态**: ✅ 已修复  

#### 问题描述
左侧任务追踪面板显示 **"46 个待完成"**，但存档中实际只定义了 21 个任务（q0 ~ q20）。这导致新手玩家看到大量未解锁的后期任务，产生困惑。

#### 根因分析
`renderQuestBox()` 函数在统计待完成任务时，使用 `!q[quest.id]` 判断。由于 `game.quests` 初始化对象只包含 `q0` ~ `q20`，而 `allQuests` 数组包含了 `q0` ~ `q50`（共 51 个任务），对于 `q21` ~ `q50` 这些后期任务，`q[quest.id]` 返回 `undefined`，`!undefined` 为 `true`，因此它们被错误地计入待完成数量。

#### 修复方案
在 `renderQuestBox()` 的统计逻辑中，增加 `hasOwnProperty` 检查，只统计存档中实际定义的任务：

```javascript
// 修复前（错误）
if (!q[quest.id]) {
    pendingQuests.push(quest);
}

// 修复后（正确）
if (q.hasOwnProperty(quest.id) && !q[quest.id]) {
    pendingQuests.push(quest);
}
```

**修复位置**: `farm_game.html` 第 9683-9687 行

#### 修复验证
修复后，对于新存档（q0~q20 全部为 false），待完成数量应为 **21**（而非 46），与实际存档定义一致。对于已推进到后期的存档，q21+ 会被正确识别为已解锁或未解锁，不再误报。

## 七、风险项与后续建议

1. **游戏时间推进速度**: 从 08:17 到 08:47（30 分钟），游戏内时间从 13:00 推进到 19:00（6 小时）。若游戏速度为 1x，此推进速度偏快。建议检查 `gameTick` 或 `advanceTime` 的调用频率，确认是否与实际速度设置一致。

2. **健康度恢复机制**: 当前健康度依赖"睡觉"和"社区建设加成"恢复，对于不进行社区建设的新手，健康度恢复极慢。建议考虑是否增加非睡眠的健康恢复途径（如日常饮食微量恢复），或优化新手引导中关于健康管理的提示。

3. **任务系统扩展**: 当前 `game.quests` 只初始化到 q20，后期任务（q21+）在 `renderQuestBox` 中硬编码。建议考虑动态任务解锁机制，避免硬编码任务与存档不同步的问题。

4. **六小时多倍速 QA 测试准备**: 当前游戏处于第 4 天早期阶段，建议继续挂机运行至更长天数（如 50+ 天），覆盖季节切换、作物完整生命周期、事件触发、技能升级、NPC 互动等系统，以验证多倍速下的稳定性。
