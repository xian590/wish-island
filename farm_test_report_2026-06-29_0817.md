## 3. 测试发现与Bug分析

### 3.1 🟡 关注项1：UI任务数与存档数据不一致

| 项目 | 描述 |
|------|------|
| **现象** | 游戏界面显示「46个待完成」任务，但 localStorage 存档中 `questCount = 21` |
| **截图证据** | 任务追踪面板显示：当前任务"整地开荒"、待办"购买种子"和"播种育苗"，底部标注"还有43个任务待完成" |
| **可能原因** | ① UI 统计了所有任务模板（包括隐藏/未解锁），而存档只统计已激活任务；② 任务系统存在前端缓存与后端数据不同步；③ 任务计数逻辑重复计算 |
| **影响评估** | 低风险——不影响游戏功能，但会造成玩家认知偏差（以为有大量任务未做） |
| **复现条件** | 新游戏开局即可复现 |
| **建议修复** | 统一前后端任务计数口径，确保 `questCount` 与 UI 显示一致；或在 UI 中区分"已解锁"和"总计"任务数 |

### 3.2 🟡 关注项2：健康度持续偏低（60/100）

| 项目 | 描述 |
|------|------|
| **现象** | 从第2天开始，健康度始终停留在 60/100，至今已持续3个游戏日（第2→3→4天） |
| **时间线** | 06:17(60) → 06:47(60) → 06:48(60) → 07:17(60) → 07:47(60) → 08:17(60) |
| **事件日志** | 已触发"吃饭花了20元，体力恢复了一些"，但未提及健康恢复 |
| **可能原因** | ① 健康恢复依赖特定行为（如休息、就医），而非吃饭；② 健康恢复事件触发概率过低；③ 健康恢复逻辑存在条件判断遗漏（例如雨天/疲劳状态影响恢复） |
| **影响评估** | 中风险——长期健康偏低可能影响游戏后期体验（如触发疾病、降低工作效率） |
| **建议验证** | 手动测试：确认健康恢复的触发条件（睡觉、休息、医疗等），并检查是否为新游戏开局保护的预期设计 |

### 3.3 🟡 关注项3：田地持续空闲，自动循环未触发

| 项目 | 描述 |
|------|------|
| **现象** | 第4天田地仍为 idle 状态，未触发任何自动播种/收获/销售行为 |
| **背景** | 修复清单中已记录"离线自动循环 harvest+plant+sell"，但当前存档中 `totalHarvest = 0`、`totalSales = 0` |
| **可能原因** | ① 自动循环功能仅在特定天数（如第5天后）或特定条件下解锁；② 当前为 normal 模式，自动循环可能仅在 easy 模式启用；③ 任务系统未引导玩家完成"整地开荒"和"购买种子"，导致自动循环的前置条件不满足 |
| **影响评估** | 中风险——新手玩家在前4天无作物收入，可能产生流失焦虑 |
| **建议验证** | 确认自动循环的触发条件，或在任务引导中增加"自动管理"的说明 |

| 优先级 | 问题 | 建议行动 |
