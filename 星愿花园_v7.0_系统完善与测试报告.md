## 七、核心功能验证

### 7.1 Vision Board 图片上传
- **流程**: 选择文件 → Canvas 压缩(800px, JPEG 0.7) → base64 → localStorage → 渲染为卡片封面
- **限制**: 12 张卡片上限，单张图片压缩后约 50-150KB，localStorage 5MB 总限制需留意
- **降级**: 无图片时回退到 emoji + 渐变背景

### 7.2 暗色模式
- **状态**: `darkMode` (布尔) + `darkModeAuto` (布尔)
- **初始化**: 如果 `darkModeAuto=true` 且 `darkMode=false`，检测 `matchMedia('(prefers-color-scheme: dark)')`
- **监听**: `mq.addEventListener('change', ...)` 实时跟随系统
- **手动切换**: Settings 页面 toggle 会覆盖自动模式（`darkModeAuto` 保持 true，但 `darkMode` 变为 true）

### 7.3 音频冥想
- **场景**: 5 个（肯定语循环、SATS 引导、财富丰盛、爱情修复、深度安眠、晨光唤醒）
- **白噪音**: `AudioBuffer` 生成粉红/棕/白噪音，通过 `BiquadFilter` 模拟 rain/fire/ocean
- **语音**: `speechSynthesis` 中文朗读，间隔 30s/分段
- **降级**: `speechSynthesis` 不可用时显示 Toast 提示；`AudioContext` suspended 时自动 `resume()`
- **已知问题**: 移动端 Safari 需要用户首次点击后才能激活 AudioContext（已处理 `resume()`）

### 7.4 跨设备同步
- **导出**: 选择数据类别 → 生成 base64 压缩文本 → 显示占位 QR 图标 + 复制文本
- **导入**: 粘贴文本 → 解析 JSON → 预览 → 确认写入 localStorage → 刷新页面
- **限制**: 无真正二维码图片生成（需引入 qrcode.js），当前以文本同步为主

## 八、性能评估（估算）

### 8.1 资源大小
| 资源 | 大小 | 压缩后 (Gzip) |
|------|------|---------------|
| index.html | 352 KB | ~70 KB |
| styles.css | 47 KB | ~10 KB |
| app.js | 570 KB | ~120 KB |
| 外部 CDN | ~150 KB | ~40 KB |
| **总下载** | ~1.1 MB | **~240 KB** |

### 8.2 关键性能指标（估算）
- **FCP (First Contentful Paint)**: ~200-400ms（取决于 CDN 和本地缓存）
- **TTI (Time to Interactive)**: ~1.5-2.5s（12,000+ 行 JS 解析）
- **LCP (Largest Contentful Paint)**: ~1.0-2.0s（依赖字体加载）
- **CLS (Cumulative Layout Shift)**: 低（固定布局，无动态插入广告）

### 8.3 优化建议
- `app.js` 570KB 仍偏大，建议按页面拆分为动态 import（`import()`）
- 字体 `display=swap` 已添加，但 `ZCOOL XiaoWei` 等中文字体文件较大（~200KB）
- 建议添加 `preload` 关键 CSS/JS
