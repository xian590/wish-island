# 宠物医生模拟器 — 版本演进计划 & QA方案

> 借鉴「李家村农耕模拟器」从初版（2万行单文件）到正式版的全部经验教训

## 一、农耕模拟器的核心教训（必须避免）

### ❌ 教训1：单文件HTML膨胀到2万行
- 农场最终 23,141 行，维护噩梦
- 65个未使用的函数拖慢解析
- 重复代码14处（音频守卫）、魔法数字218次

**→ 宠物医生对策**：从v0.3开始就拆模块
```
js/
  core/          // 游戏状态、存档、常量
  scenes/        // 场景管理、切换
  patients/      // 病例数据、病症库
  minigames/     // 迷你游戏独立文件
  ui/            // 对话框、状态栏、道具栏
  utils/         // 工具函数（避免重复）
```

### ❌ 教训2：存档兼容性崩溃
- 旧存档缺少新字段 → `fixSaveData` 越来越复杂
- 空catch块静默吞错误 → 玩家存档损坏不知道原因
- localStorage超出配额 → 游戏直接崩溃

**→ 宠物医生对策**：
- 版本号写入存档 `saveVersion: 3`
- 每次加载时版本检测 + 自动迁移
- localStorage全量try-catch + 降级到内存存档

### ❌ 教训3：长时间运行内存泄漏
- 20个setTimeout只清4个
- innerHTML频繁触发重排
- 箭头函数闭包持有DOM引用

**→ 宠物医生对策**：
- 所有定时器统一管理，页面卸载前全清
- 优先用textContent，少用innerHTML
- 场景切换时销毁不再需要的DOM引用

### ❌ 教训4：没有自动化测试
- 全靠手动点击验证，漏掉边缘情况
- 六小时挂机测试才发现定时器累积bug

**→ 宠物医生对策**：
- v0.5起加入自动化测试脚本（模拟点击、验证状态）
- 每个版本必须通过"冒烟测试清单"

### 2. 错误处理（绝不静默吞错误）
```javascript
// ❌ 农场踩过的坑
try {
    saveGame();
} catch(e) {}  // 静默失败，玩家不知道存档坏了

// ✅ 正确做法
try {
    saveGame();
} catch(e) {
    console.error('[SAVE ERROR]', e);
    showToast('存档失败：' + e.message, 'error');
    // 降级到内存存档
    memoryBackup = JSON.stringify(gameState);
}
```

### 3. 存档版本迁移
```javascript
const SAVE_VERSION = 3;

function migrateSave(data) {
    const v = data.saveVersion || 0;
    
    if (v < 1) {
        // v0.2 → v0.3: 新增inventory字段
        data.inventory = { medicine: 5, herb: 3, bandage: 2 };
    }
    if (v < 2) {
        // v0.3 → v0.4: 新增npcBonds字段
        data.npcBonds = {};
    }
    if (v < 3) {
        // v0.4 → v0.5: 新增achievements字段
        data.achievements = [];
    }
    
    data.saveVersion = SAVE_VERSION;
    return data;
}
```

### 4. 内存管理
```javascript
// 定时器统一注册
const activeTimers = new Set();

function safeSetTimeout(fn, delay) {
    const id = setTimeout(() => {
        activeTimers.delete(id);
        fn();
    }, delay);
    activeTimers.add(id);
    return id;
}

// 场景切换前全清
function clearAllTimers() {
    activeTimers.forEach(id => clearTimeout(id));
    activeTimers.clear();
}

// 页面卸载前全清
window.addEventListener('beforeunload', clearAllTimers);
```
