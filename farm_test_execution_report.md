## 二、已确认的 Bug（5项，已静态验证）

### 🔴 Bug 1：田地等级系统完全未实现（严重）

- **描述**: `SKILL_EFFECTS` 中作物熟悉度 Lv2/Lv5/Lv8 分别解锁"进阶档/精英档/终极档"田地，加成分别为1.15/1.35/1.6倍。
- **实际代码**: `initGame` 和 `rentNewField` 中所有田地 `level: '新手档'`，代码中没有任何函数修改 `field.level`。
- **影响**: 所有田地永远只有1.0倍产量，技能解锁的高级田档位完全无效。玩家花大量时间升级技能，但收益归零。
- **验证位置**: `initGame` (~5640), `rentNewField` (~15025), `harvestCrop` (~14551)
- **修复建议**: 在 `rentNewField` 和 `addSkillExp`/`levelUpSkill` 中添加田地等级升级逻辑，根据 `cropFamiliarity.level` 设置新田等级。

### 🟡 Bug 2：事件限制为 `< 2` 而非 `≤ 1`（中等）

- **描述**: 用户期望每天最多1个随机事件。
- **实际代码**: `if ((game.eventsToday || 0) < 2)` 允许每天最多2个随机事件，加上 pending 故事事件可能更多。
- **位置**: `nextDay` (~7267)
- **修复建议**: 将 `< 2` 改为 `< 1`（或 `=== 0`）。

### 🟡 Bug 3：日常活动配置与实现不一致（中等）

- **描述**: `DAILY_ACTIVITIES` 配置中 `exercise.dailyLimit = 2`, `visit.dailyLimit = 3`。
- **实际实现**: `doExercise` 限制为1次，`doChat` 限制为2次，`doFish` 限制为1次，`doRead` 限制为1次。
- **位置**: `DAILY_ACTIVITIES` (~3141), `doExercise` (~6107), `doChat` (~6118)
- **修复建议**: 统一配置与实现，或删除配置对象直接由函数控制。

### 🟡 Bug 4：建筑建造限制未执行（中等）

- **描述**: `game.buildingsToday` 存在每日建造计数，但 `buildBuilding()` 函数未检查此限制。
- **影响**: 理论上可以一天内无限建造，只是每天重置计数器。
- **位置**: `buildBuilding` (~13400), `nextDay` (~7315)
- **修复建议**: 在 `buildBuilding` 开头添加 `if (game.buildingsToday.count >= 1) return;`。

### 🟡 Bug 5：模式UI描述与代码数值不符（中等）

- **描述**: HTML 中 easy/normal/hard 的初始健康/体力显示为 100/120，但 `initGame` 中实际为 200/200（easy/normal）和 220/220（hard）。
- **位置**: HTML mode-stats (~1864-1898) vs `initGame` (~5620-5623)
- **修复建议**: 修改 HTML 中的初始值显示，或修改 `initGame` 中的初始值以匹配UI描述。

### 🟢 Bug 6：加工解锁条件文字歧义（轻微）

- **描述**: PROCESSING_DATA 中 `unlockCondition: '作物熟悉度Lv2或攒够2000块钱'`。
- **实际代码**: `isProcessingUnlocked` 检查 `cropSkillLevel >= 2 || totalAssets >= 2000`（总资产 = 现金 + 库存作物价值）。
- **修复建议**: 修改文字描述为"总资产达2000"以避免玩家误解。

## 三、执行测试时发现的新问题（动态验证）

### ⚠️ 问题 1：autoPlay 无法精确停止（测试阻碍）

- **现象**: 通过 `clearInterval` 和设置 `gameSpeed=0` 无法完全停止游戏自动推进。游戏仍在后台运行，导致天数从1自动推进到17。
- **影响**: 无法按测试方案逐日精确控制操作，所有手动验证步骤被autoPlay打乱。
- **根因**: 游戏可能使用 `setTimeout` 递归调用 `gameTick`，而非单一的 `setInterval`。清除interval后，递归的timeout仍然继续触发。
- **建议**: 测试前需要彻底重构autoPlay的停止机制，或提供一个暂停/恢复API。

### ⚠️ 问题 2：NPC 好感度始终为 0（严重）

- **现象**: 游戏运行17天，所有NPC好感度始终为 `{}`（空对象），没有任何NPC有好感度记录。
- **影响**: NPC系统完全失效：无法触发NPC事件、无法雇佣工人、无法获得NPC里程碑加成。
- **根因待查**: 可能原因：
  1. `autoPlayTick` 中的自动串门没有正确调用 `chatWithNpc` 或 `addNpcFavor`
  2. `npcs` 对象的初始化或更新逻辑有bug
  3. `doChat` 函数中没有正确更新 `game.npcs`
- **需要**: 检查 `doChat` 和 `autoPlayTick` 中的NPC好感更新逻辑。

### ⚠️ 问题 3：随机事件从未触发（严重）

- **现象**: 游戏运行17天，`eventHistoryCount` 始终为 0，没有任何随机事件记录。
- **影响**: 事件系统完全失效，游戏缺少随机性和趣味性，经济/健康/心情等系统无法通过事件验证。
- **根因待查**: 可能原因：
  1. `triggerRandomEvent` 函数内部有错误导致提前返回
  2. 事件概率计算错误（如 `game.disasterRate` 为0）
  3. 事件触发条件（如需要有作物）在初期不满足
- **需要**: 检查 `triggerRandomEvent` 函数和事件触发概率计算。

### ⚠️ 问题 4：资金归零与健康归零（严重）

- **现象**: 游戏运行17天后，资金=0，健康=0。
- **影响**: 玩家进入破产+濒死状态，无法继续正常游戏。
- **根因待查**: 可能原因：
  1. autoPlay的出售策略有问题，导致资金流失
  2. 建筑/种子购买消耗过大，收入不足以覆盖
  3. 健康消耗事件频繁触发但没有恢复机制
  4. 夜间事件导致健康持续下降
- **需要**: 检查经济收支平衡和健康恢复机制。

### ⚠️ 问题 5：初始资金与代码不符（待确认）

- **现象**: Normal 模式初始资金为 1900 元，但代码分析中 `configs.normal.money = 1500`。
- **可能原因**: 代码已被修改但分析时未刷新，或 `initGame` 中有额外奖励逻辑。
- **需要**: 重新确认代码中Normal模式的初始资金配置。
