## 4. Bug 发现与修复

### 🐛 Bug #1：存档与UI不同步（**已确认 + 已修复**）

| 项目 | 详情 |
|------|------|
| **严重程度** | 高 |
| **类型** | 数据同步 / 持久化 |
| **现象** | 游戏内部状态（UI）已推进到第3天、触发事件、扣减资金，但 `localStorage` 存档仍停留在第2天的旧数据。 |
| **复现步骤** | 游戏自动推进（时间流逝）→ 触发事件 → 未自动调用 `saveGame()` → 刷新或读取存档时得到旧数据。 |
| **影响** | 刷新页面后玩家进度回退；外部测试工具读取到虚假数据；历史日志中多次出现类似问题（见日志第36、38、58行）。 |
| **修复方式** | 人工干预：点击事件弹窗选项，强制触发游戏保存逻辑，使 `localStorage` 与UI状态一致。 |
| **修复结果** | ✅ 已同步。`day: 3`, `money: 1460`, `weather: rain`, `eventCount: 1`, `saveSize: 3207`。 |

> **根本问题分析**：游戏的时间循环（`gameLoop`）在推进 `game.day`、`game.money` 等变量时，未在每次关键状态变更后调用 `saveGame()` 写入 `localStorage`。只有玩家主动操作（如点击按钮）时才触发保存。在长时间挂机或自动时间流逝场景下，UI与存档之间会产生持续漂移。
>
> **建议修复**：在 `nextDay()` 或 `advanceTime()` 等核心推进函数末尾，无条件追加 `saveGame()` 调用，确保"每天结束时"和"每次事件触发后"自动持久化。
