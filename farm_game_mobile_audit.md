# 农场游戏移动端全面扫描报告

**扫描文件**: `farm_game.html`  
**总问题数**: 48

---

## 高优先级问题 (11条)

### 1. CSS .event-log (第897-914行, 第2073-2087行)

**严重程度**: 高

**问题描述**: 移动端事件日志使用 `position: fixed` 固定在底部 (`bottom: 120px`)，但 bottom 定位本身未考虑 iOS 安全区域。在带 Home 指示条的 iPhone 上，日志区域可能被遮挡或遮挡底部导航。

**用户影响**: 用户可能看不到重要的事件日志，或日志遮挡底部导航栏按钮。

**修复建议**: 添加 `bottom: calc(120px + env(safe-area-inset-bottom))`，同时增大内容面板 `padding-bottom`。

---

### 2. CSS .modal (第1074-1092行, 第2088-2104行)

**严重程度**: 高

**问题描述**: 模态框在移动端设置了 `max-height: 85vh`，但标题和按钮区域是固定高度的，如果内容过长，在小屏幕上内容可能被压缩到无法阅读。`countdown-modal` 等包含大量信息的弹窗问题更明显。

**用户影响**: 用户无法完整阅读模态框内容，特别是任务进度弹窗。

**修复建议**: 确保 `modal-content` 有足够的最小高度，或允许小屏幕上模态框全屏显示。

---

### 3. CSS .mode-card + 顶部状态栏 (第161-177行, 第1897-2148行)

**严重程度**: 高

**问题描述**: 模式选择卡片桌面端固定宽度 420px。游戏主界面顶部状态栏在 375px 屏幕上内容过多：`stats-right` 隐藏了第4、5个 `stat-item`（第2022-2025行），但剩余内容仍然可能被压缩或溢出。

**用户影响**: 小屏幕用户无法看到完整的资源统计（玉米、大豆库存被隐藏），严重影响游戏决策。

**修复建议**: 在 375px 及以下添加更细粒度的媒体查询，进一步精简顶部状态栏，或允许横向滑动查看隐藏的统计项。

---

### 4. CSS .sidebar (第399-405行, 第1902-1912行)

**严重程度**: 高

**问题描述**: 移动端 `sidebar` 固定在底部 (`position: fixed; bottom: 0; z-index: 1000`)，高度不固定（取决于任务追踪器）。`content-panel` 的 `padding-bottom: 250px` 可能不足以覆盖 `sidebar` 高度，导致内容被遮挡。

**用户影响**: 用户无法看到或点击页面底部内容（如田地卡片的底部操作按钮）。

**修复建议**: 将 `sidebar` 高度固定或设置 `max-height`，确保 `content-panel` 的 `padding-bottom` 始终大于 `sidebar` 高度 + 安全区域。

---

### 5. CSS .speed-btn-top (第297-323行, 第1984-1987行)

**严重程度**: 高

**问题描述**: 桌面端 `speed-btn-top` 的 `padding: 4px 10px`，移动端进一步减小到 `padding: 3px 6px`。计算高度约: font-size 11px + padding 上下 6px + border 2px = 约 19px，远小于 44px 推荐触摸目标尺寸。

**用户影响**: 用户难以准确点击时间流速按钮，容易误触旁边暂停按钮。

**修复建议**: 将触摸目标尺寸增加到至少 44px × 44px，通过增大 padding 或设置 `min-width`/`min-height`。

### 6. CSS .stat-item (第368-381行, 第1976-1978行)

**严重程度**: 高

**问题描述**: `stat-item` 在桌面端 `padding: 4px 8px`，`font-size: 14px`，移动端 `font-size: 12px`。整个元素高度可能不足 44px。

**用户影响**: 顶部状态栏的各个统计项难以点击。

**修复建议**: 增大 `stat-item` 的 padding，确保最小高度 44px。

### 7. CSS 缺少暗黑模式支持

**严重程度**: 高

**问题描述**: 没有 `@media (prefers-color-scheme: dark)` 规则。如果用户在系统设置中开启了暗黑模式，页面仍显示浅色背景，在暗光环境下刺眼。

**用户影响**: 夜间或暗光环境下使用体验差，可能导致眼睛疲劳。

**修复建议**: 添加暗黑模式媒体查询，为 body、panel、card 等元素提供暗色主题。

### 8. CSS 缺少横屏(landscape)媒体查询

**严重程度**: 高

**问题描述**: 代码中没有 `@media (orientation: landscape)` 规则。在横屏模式下，底部固定导航栏占据了宝贵的垂直空间，而横向空间可能足够容纳侧边栏布局。

**用户影响**: 横屏模式下用户体验差，底部导航栏占用大量游戏区域，内容显示区域过小。

**修复建议**: 添加横屏媒体查询，在横向空间足够时恢复桌面端的侧边栏布局。

### 9. HTML 使用 div/span 代替 button (多处)

**严重程度**: 高

**问题描述**: `settings-btn`（第1436-1457行）、`event-log-expand-btn`（第965-978行）、`sidebar-tab`（第413-445行）等使用 `div` 模拟按钮。这些元素没有 `button` 的默认键盘行为（Enter/Space 激活）。

**用户影响**: 键盘用户无法使用 Tab 键导航到这些元素，也无法使用 Enter 或 Space 键激活它们。屏幕阅读器不会识别为按钮。

**修复建议**: 将所有模拟按钮替换为原生 `button` 标签，或添加 `role="button"`、`tabindex="0"`，并监听 `keydown` 事件。

---

### 10. HTML 设置面板 range 输入 (第2926行, 第2936行)

**严重程度**: 高

**问题描述**: 音量滑块使用 `input type="range"`，但没有任何标签说明当前值。用户只能通过视觉大致判断音量大小。

**用户影响**: 用户无法准确设置音量，特别是需要精确调整时。

**修复建议**: 添加音量数值显示标签（如 "30%"），或使用更大的滑块控制。

---

### 11. HTML/CSS 缺少 aria-label 和 role 属性

**严重程度**: 高

**问题描述**: 整个页面几乎没有使用 `aria-label` 或 `role` 属性。`sidebar-tab`、`mode-card`、`settings-btn`、`quick-btn` 等交互元素缺乏语义化标签。

**用户影响**: 屏幕阅读器用户无法理解界面结构，无法操作游戏。例如，屏幕阅读器只会读出 "div 田地管理" 而不是 "按钮 田地管理"。

**修复建议**: 为所有交互元素添加适当的 `role` 属性（如 `role="button"`、`role="tab"`）和 `aria-label`/`aria-labelledby`。

**严重程度**: 中

**问题描述**: 页面没有为输入框添加 focus 事件处理器来确保输入框在键盘弹出时可见。在 iOS Safari 中，固定定位的底部导航栏和事件日志可能会遮挡聚焦的输入框。

**用户影响**: 用户点击输入框后，键盘弹出可能遮挡输入框，无法看到自己输入的内容。

**修复建议**: 添加 input/textarea 的 focus 事件监听，使用 `scrollIntoView` 或 `window.scrollTo` 确保输入框可见。考虑使用 `scroll-padding-bottom`。

### 12. CSS prefers-reduced-motion 支持不完整

**严重程度**: 中

**修复建议**: 在 `prefers-reduced-motion` 中禁用所有过渡和动画效果，包括 `::before`/`::after` 伪元素的动画。

### 13. CSS z-index 层级混乱

**严重程度**: 中

**问题描述**: z-index 值分布不均匀：`story-container(3000)`, `select-container(2500)`, `modal-overlay(3500)`, `quick-actions(1500)`, `float-countdown-btn(2000)`, `toast(3000)`, `sidebar-mobile(1000)`, `event-log(1001)`, `quick-actions-mobile(2002)`, `file-warning(100)`, 迁移 banner(99999)。层级设置不系统，迁移 banner(99999) 远高于 `modal-overlay(3500)`。

**用户影响**: 某些情况下弹窗可能被意外元素遮挡，或元素层级关系不符合预期。

**修复建议**: 建立统一的 z-index 层级系统（如使用 CSS 变量），按逻辑分组：基础内容(0-100)、导航(100-200)、浮层(300-500)、模态框(1000+)。

### 24. JS 游戏主循环 (setInterval)

**严重程度**: 中

**问题描述**: 游戏使用 `setInterval` 驱动主循环，如果游戏逻辑复杂或设备性能低，可能导致定时器堆积，造成界面卡顿。
