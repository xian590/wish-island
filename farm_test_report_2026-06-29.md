## 3. 异常与 Bug 检测

### 🔴 严重异常：健康度过夜不恢复

| 项目 | 详情 |
|------|------|
| **现象** | 健康度从 60/100 持续不变，即使过了一夜（游戏内时间从第 3 天 18:00 推进到第 4 天 07:00） |
| **历史记录** | 06:17 记录健康 60/100 → 06:47 健康 60/100 → 06:48 健康 60/100 → 07:17 健康 60/100 → **07:47 健康 60/100** |
| **预期行为** | 睡觉/过夜应该恢复一定量的健康值（例如 +10~20） |
| **实际行为** | 健康值无任何变化，持续锁定在 60 |
| **影响评估** | 高。健康度持续偏低可能导致后续事件触发受限、角色状态恶化，影响游戏体验 |
| **可能原因** | 1. 健康恢复逻辑未绑定到 `day` 或 `time` 变化事件<br>2. 恢复条件过于苛刻（如需要特定食物/道具）<br>3. 恢复公式存在 bug（如恢复值计算为 0 或负值）<br>4. 存档中的健康值被覆盖逻辑错误 |
| **建议修复** | 1. 检查 `nextDay()` 或 `advanceTime()` 中是否包含 `health += recoverAmount` 逻辑<br>2. 确认健康恢复是否依赖 `sleep()` 或 `rest()` 的显式调用<br>3. 添加调试日志追踪健康值变化路径 |

### 🟡 需关注：第 4 天仍未种植作物

| 项目 | 详情 |
|------|------|
| **现象** | 游戏已进行至第 4 天，唯一的一块田地仍处于空闲状态（idle） |
| **影响评估** | 中。新玩家可能不知道如何触发种植，或缺少引导提示 |
| **可能原因** | 1. 玩家（测试环境）未主动操作<br>2. 缺少新手引导提示 "点击田地种植"<br>3. 种植交互逻辑存在可发现性问题 |
| **建议** | 在 UI 中添加新手引导提示，当田地空闲且天数 >= 2 时提示玩家种植 |

### 🟡 需关注：事件触发频率偏低

| 项目 | 详情 |
|------|------|
| **现象** | 4 天仅触发 1 个事件（"大喇叭广播"） |
| **历史调整** | 此前已将事件概率从 0.02/0.025 提高至 0.06/0.08（2-4 倍） |
| **可能原因** | 1. 事件触发条件仍然过于严格（如 `minDay`/`maxDay` 限制）<br>2. 事件检查频率不足（如仅在特定时间点检查）<br>3. 随机数生成器种子问题 |
| **建议** | 进一步放宽事件触发条件，或增加每日强制检查机制 |

## 5. 修复建议（按优先级排序）

### P0 - 立即修复
1. **健康度恢复逻辑 Bug**
   - 检查 `nextDay()` 或 `advanceTime()` 函数中是否包含健康恢复代码
   - 确认 `health` 的恢复条件是否过于苛刻（如需要 `sleep()` 显式调用）
   - 添加调试日志：`console.log('health before:', health, 'after:', health + recovery)`
   - 建议：如果 `time` 推进至次日早晨（如 06:00-08:00），自动触发健康恢复

### P1 - 尽快修复
2. **事件触发频率优化**
   - 检查事件概率公式是否被其他条件覆盖
   - 考虑添加每日保底事件机制（每天至少 1 个随机事件）

3. **新手种植引导**
   - 当 `day >= 2` 且 `fields` 全为 `idle` 时，显示浮动提示 "点击田地开始种植"

### P2 - 建议改进
4. **健康度 UI 预警**
   - 当健康值低于 60% 时，健康值显示变为红色或添加闪烁动画

5. **存档健康检查**
   - 在存档加载时验证关键数值范围（如 `health` 应在 `0~maxHealth` 之间）
