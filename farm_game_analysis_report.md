## 二、数值系统分析

### 2.1 作物收益分析

| 作物 | 种子成本 | 产量(斤) | 成熟天数 | 售价(元/斤) | 总收入 | 日净利润 | 投入产出比 | 评价 |
|------|---------|---------|---------|------------|--------|---------|-----------|------|
| 早稻(春) | 60 | 200 | 5 | 3.0 | 600 | **108元/天** | 10.0x | 优秀 |
| 晚稻(夏) | 70 | 180 | 4 | 3.0 | 540 | **117.5元/天** | 7.7x | 优秀 |
| 春薯 | 80 | 750 | 10 | 1.5 | 1125 | **104.5元/天** | 14.1x | 优秀 |
| 秋薯 | 90 | 550 | 8 | 1.5 | 825 | **91.9元/天** | 9.2x | 优秀 |

**分析结论**：所有作物日净利润都在90-120元之间，投入产出比7-14倍，非常健康。但缺少**日收益排行榜**让玩家直观比较。

### 2.2 加工利润率分析

| 加工品 | 输入(斤) | 输出(斤) | 输入价值 | 输出价值 | 利润率 | 评价 |
|--------|---------|---------|---------|---------|--------|------|
| 糙米 | 10稻 | 6.5糙米 | 30元 | 39元 | **30%** | 偏低 |
| 红薯干 | 10薯 | 4干 | 15元 | 24元 | **60%** | 高利润 |
| 精米 | 10糙米 | 8精米 | 60元 | 81.6元 | **36%** | 合理 |
| 米酒 | 8精米 | 4酒 | 81.6元 | 106元 | **30%** | 偏低 |

**分析结论**：加工利润率30-60%，但糙米和米酒只有30%。对比星露谷酿酒流（小桶陈酿后收益翻3-5倍），我们的加工深度不够。**建议深加工品利润率提升到50-100%**。

### 2.3 体力经济分析

| 项目 | 数值 | 分析 |
|------|------|------|
| 初始体力 | 100（亚健康） | 合理，符合设定 |
| 最大体力 | 100（正常模式） | 合理 |
| 每小时恢复 | 13（新手房） | 一天恢复312点 |
| 单次操作消耗 | 10-20点 | 典型值15点 |
| 每天可操作次数 | 312/15 ≈ **20次** | 严重过多！ |
| 每天吃饭扣钱 | 20元 | 合理 |

**核心问题**：一天恢复312点体力，但单次操作只需15点，玩家每天可以操作20次。这导致：
- 玩家在第44天春季就已经解锁所有内容，因为每天操作次数太多
- 没有"体力管理"的策略深度
- 睡觉的必要性被削弱

**对标星露谷**：星露谷每天270体力，单次操作2-8点，但工具升级后才降低消耗。玩家需要精心规划每天做什么。**建议我们的单次操作消耗提升到20-30点，或降低恢复率**。

| 房屋 | 升级成本 | 日租金 | 每小时恢复 | 日总恢复 | 回本周期 |
|------|---------|--------|-----------|---------|---------|

| 进阶房 | 600 | 180/季≈3/天 | 17 | 408 | 8天 |

**分析**：回本周期8-42天，非常合理。但**缺少房屋升级带来的视觉反馈**（如星露谷的农场外观变化）。

### 2.5 季节系统分析

| 对比项 | 李家村 | 星露谷 | 放置标准 |
|--------|--------|--------|---------|
| 季节长度 | **60天** | 28天 | 通常7-14天 |
| 作物跨季 | 枯萎 | 枯萎 | 通常不枯萎 |
| 季节事件 | 少量 | 丰富 | 周期性事件 |

**核心问题**：60天太长。玩家在第44天春季就感觉没事做。放置游戏标准每3-5分钟解锁一个新内容，60天意味着玩家需要等待**86400分钟**才能进入下一季。**建议缩短到28-30天**。

## 四、核心问题总结（13个）

### P0 - 必须立即修复（4个）
1. **缺少离线收益系统** - 放置游戏的核心，没有离线收益就不是放置游戏
2. **缺少重置/转生（Prestige）系统** - 长线动力不足，玩家达到终点后无目标
3. **季节60天太长** - 玩家在第44天就感觉没事做，放置标准28天
4. **经济曲线线性增长** - 成本应该指数增长，制造"前期快、后期慢"的放置感

### P1 - 高优先级（5个）
5. **体力恢复远超消耗** - 每天可操作20次，太多。应限制到8-12次/天
6. **加工系统只有2-3层深度** - 缺少星露谷式的5层加工链（稻→糙米→精米→米酒→陈酿）
7. **社交系统只有简单好感度** - 缺少事件链、结婚、专属任务
8. **缺少探索内容** - 没有矿洞、地下城、随机地图
9. **里程碑解锁频率低** - 每3-5天一个，放置标准每3-5分钟应有小反馈

### P2 - 中优先级（3个）
10. **缺少技能专精分支** - 玩家没有选择感（如星露谷的5技能×2分支）
11. **NPC事件硬编码** - 缺少动态事件生成系统
12. **宠物系统功能浅** - 只有喂食和抚摸，缺少采集/辅助功能

### P3 - 低优先级（1个）
13. **天气对作物只有简单枯萎** - 缺少实时影响（如雨天不用浇水、晴天产量+10%）

## 五、具体改进建议

### 5.1 离线收益系统（P0）

```
设计：离线收益 = 工人产出 + 加工坊产出 + 作物自然生长
- 离线最多计算8小时（防止玩家离开太久）
- 回来显示弹窗："你离开了X小时，工人帮你干了Y活，赚了Z元"
- 离线时体力恢复到满（但不超过max）
```

### 5.2 重置/转生系统（P0）

```
设计：达到第2年（240天）后可以"衣锦还乡"
- 重置：所有田地、金钱、库存清零
- 保留：声望、已解锁的科技知识（变为"知识碎片"）
- 奖励：永久加成（如作物产量+10%、加工速度+10%）
- 每次重置加成叠加，制造"越转越强"的放置感
```

### 5.3 季节缩短到28天（P0）

```
修改：game.day > 60 → game.day > 28
影响：玩家每年可以经历4个完整季节，保持新鲜感
补偿：作物生长天数等比例缩短（如早稻5天→3天）
```

### 5.4 经济指数增长（P0）

```
修改：田地租金、房屋升级、仓库扩建成本按指数增长
- 第2块田：100元
- 第3块田：200元（2x）
- 第4块田：400元（2x）
- 第5块田：800元（2x）

当前：第3块田100元，第4块200元，已经是指数！但需更明显
建议：第2块50，第3块100，第4块200，第5块400，第6块800...
```

### 5.5 体力消耗提升（P1）

```
修改：单次操作消耗提升到20-30点
- 整地：25（当前15）
- 育苗：20（当前12）
- 移栽：25（当前18）
- 收割：30（当前20）
- 钓鱼：25（当前15）
- 采集：25（当前15）

目标：每天可操作8-10次，让玩家有"今天做什么"的选择感
```

### 5.6 加工深度扩展（P1）

```
新增第4层加工：
- 米酒 → 陈酿米酒（需要3季游戏时间，售价翻3倍）
- 红薯粉条 → 酸辣粉（需要调料，售价翻2倍）
- 鱼干 → 烟熏鱼（需要木材，售价翻2倍）

新增第5层加工：
- 陈酿米酒 → 珍藏酒（需要5季，售价翻5倍）
```

### 5.7 矿洞探索系统（P1）

```
设计：后山矿洞（30层）
- 每天可下矿1次，消耗30体力
- 每层有概率获得矿石、宝石、稀有种子
- 矿石用于升级工具（减少体力消耗）
- 宝石用于科技研究（替代农技经验）
- 层数越深，奖励越好，但也越危险（可能遇到塌方，扣健康）
```

### 5.8 技能专精分支（P2）

```
作物熟悉度Lv5时选择：
- 分支A：农耕大师（产量+20%）
- 分支B：育种专家（解锁稀有种子）

田间管理Lv5时选择：
- 分支A：灌溉专家（减少浇水次数）
- 分支B：除虫专家（减少虫害损失）
```

### 5.9 天气实时影响（P2）

```
雨天：
- 作物自动浇水（省体力）
- 但钓鱼收益-30%（鱼不上钩）
- 采集无法进行（山路泥泞）

晴天：
- 作物产量+10%
- 钓鱼收益+20%

大风：
- 有概率吹倒作物（损失10%产量）
```

### 5.10 里程碑密度提升（P1）

```
小里程碑（每3-5分钟）：
- 每完成一次操作就弹出"+经验"
- 每卖出一次作物就显示"累计收入XXX元"
- 每过一天就显示"第X天 - 成就提示"

中里程碑（每30分钟）：

大里程碑（每3-5天）：
```

## 七、总结

**当前状态**：游戏基础框架完整，但处于"有框架无灵魂"的阶段。13,575行代码实现了大量系统，但缺少放置游戏的核心机制（离线收益、重置、指数增长）。

**核心矛盾**：用户要求"不要自动，全部手动"，但放置游戏的核心就是"部分自动化+离线收益"。这需要在"手动体验"和"放置乐趣"之间找到平衡：
- 保持关键操作手动（种植、加工、社交）
- 引入自动化辅助（工人、离线收益、洒水器）
- 用"选择"替代"纯手动"（今天做什么的选择，而不是什么都做）

**短期目标**：修复11个bug + 实现离线收益和季节缩短
**中期目标**：加入重置系统、矿洞探索、技能分支
**长期目标**：加工深度扩展到5层、社交系统完整、天气系统实时
